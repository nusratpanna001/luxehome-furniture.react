## Data Flow

### Authentication Flow
```
Login/Register
    ↓
AuthContext.login/register
    ↓
Mock/Real API Call
    ↓
Store token & user in localStorage
    ↓
Update AuthContext state
    ↓
Navigate to /dashboard
```

### Protected Route Flow
```
User navigates to protected route
    ↓
ProtectedRoute checks isAuthenticated
    ↓
If not authenticated → Redirect to /login
    ↓
If authenticated, check role
    ↓
If role matches → Render page
    ↓
If role doesn't match → Show access denied
```

### CRUD Flow (Example: Products)
```
User clicks "Add Product"
    ↓
Modal opens with ProductForm
    ↓
User fills form & submits
    ↓
Form validates via Zod schema
    ↓
API call (mock or real)
    ↓
Optimistic UI update
    ↓
Show success toast
    ↓
Close modal & refresh list
```

## Key Features Implementation

### Mock Mode
- Enabled via `VITE_USE_MOCK=true` in `.env`
- Mock data in `src/lib/mockData.js`
- Mock service implements same API as real backend
- All CRUD operations work without backend

### RBAC (Role-Based Access Control)
- Roles defined in `src/lib/constants.js`
- Helper functions in `src/lib/rbac.js`
- Sidebar dynamically filters routes by role
- ProtectedRoute enforces role requirements

### Form Validation
- Zod schemas in `src/lib/formSchemas.js`
- React Hook Form for state management
- Inline error messages
- Disabled submit until valid

### Responsive Design
- Mobile-first Tailwind utilities
- Sidebar drawer on mobile
- Responsive grids and tables
- Touch-friendly buttons

### Performance Optimizations
- Lazy loading pages with React.lazy()
- Code splitting by route
- Debounced search inputs
- Memoized computed values
